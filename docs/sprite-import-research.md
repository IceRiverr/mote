# 精灵图导入流程研究

## 当前问题分析

### 界面截图观察

当前的导入界面存在以下问题：

1. **布局不合理**
   - 导入弹窗从顶部展开，占据整个宽度
   - 遮挡了画布区域的引导提示
   - 视觉重心被分散

2. **信息层级混乱**
   - 四个导入模式平铺展示，没有主次之分
   - W/H/M/S 参数对所有模式都可见（但只对网格模式有用）
   - 文件选择器放在模式选择之后，但应该先选文件再决定如何处理

3. **交互流程反直觉**
   - 用户需要：先选模式 → 再选文件 → 再设置参数 → 导入
   - 更符合直觉的流程：先选文件 → 系统检测类型 → 显示相关参数 → 预览 → 导入

4. **缺乏反馈**
   - 选择文件后没有预览
   - 没有切片预览
   - 导入前看不到最终效果

## 其他编辑器的导入设计

### 1. Unity (Sprite Editor)

**导入流程：**
```
1. 在 Project 窗口右键 → Import New Asset
2. 选择图片文件
3. 在 Inspector 设置 Texture Type = Sprite (2D and UI)
4. 设置 Sprite Mode = Multiple (如果是图集)
5. 点击 Sprite Editor 打开编辑器
6. 在 Sprite Editor 中点击 Slice 进行切片
```

**特点：**
- 资源导入和切片分离
- 切片时有多种模式（自动、网格、按单元格）
- 实时预览切片结果

### 2. Tiled (瓦片图编辑器)

**导入流程：**
```
1. 新建图块集 (New Tileset)
2. 选择图片来源（图片文件）
3. 弹出切片对话框：
   - 类型：基于整张图片 / 基于多个图片
   - 嵌入地图 / 独立图块集
4. 如果选择切片：
   - 选择模式：网格 / 按瓦片宽度 / 按瓦片高度
   - 设置参数
5. 实时预览切片结果
```

**特点：**
- 向导式流程，一步一步引导
- 选择图片后才显示切片选项
- 实时预览

### 3. Aseprite (像素画编辑器)

**导入 Sprite Sheet：**
```
1. File → Import Sprite Sheet
2. 选择图片文件
3. 弹出导入对话框：
   - 切片方向：水平 / 垂直 / 行列
   - 设置切片尺寸或数量
   - 实时预览切片网格
4. 确认导入
```

**特点：**
- 导入前有可视化预览
- 可以直接在图片上看到切片网格
- 参数调整实时反映在预览上

### 4. TexturePacker (专业精灵图工具)

**导入流程：**
```
1. 拖拽图片到软件
2. 自动检测透明区域并切片
3. 在右侧设置：
   - 算法（网格、最大矩形、多边形等）
   - 参数（间隙、内边距等）
4. 实时预览打包结果
5. 导出 JSON/PNG
```

**特点：**
- 拖拽即导入
- 自动检测
- 强大的实时预览

### 5. Godot 引擎

**导入流程：**
```
1. 将图片放入项目文件夹（自动导入）
2. 在 FileSystem 点击选中图片
3. 在 Import 标签页设置：
   - 类型：Texture / Sprite Atlas
   - 如果是 Atlas：设置切片参数
4. 点击 Reimport
```

**特点：**
- 自动导入
- 按需重新导入
- 参数化配置

### 方案 2: 简化弹窗 + 智能检测

**流程：**
```
1. 点击"导入"按钮
2. 选择文件（可多选）
3. 系统检测文件类型：
   - 如果是 PNG/JPG：默认网格模式
   - 如果有同名 JSON/XML：自动匹配
4. 弹出配置弹窗，带实时预览
5. 确认导入
```

**弹窗设计：**
```
┌─────────────────────────────────────────────────────────────┐
│ 导入: tiny-dungeon.png              [网格 ▼]     [✕]       │
├─────────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ ┌───────────────┐  │
│ │                                     │ │ 导入方式       │  │
│ │         [图片预览 + 切片网格]        │ │               │  │
│ │                                     │ │ ○ 网格         │  │
│ │         显示红框网格覆盖层           │ │ ● JSON        │  │
│ │                                     │ │ ○ XML          │  │
│ │                                     │ │ ○ 散图         │  │
│ └─────────────────────────────────────┘ │               │  │

```

### 方案 3: 拖拽导入（长期）

```
1. 用户从文件管理器拖拽图片到编辑器
2. 自动打开导入配置弹窗
3. 显示预览和参数
4. 确认导入
```

## 立即可做的改进

### 1. 弹窗改为居中模态对话框
- 不再从顶部展开
- 居中显示，背景半透明遮罩
- 更好的视觉焦点

### 2. 流程改为：文件 → 模式 → 参数
- 第一步：选择文件
- 第二步：选择导入方式（带智能推荐）
- 第三步：设置参数 + 预览

### 3. 参数动态显示
- 只有网格模式才显示 W/H/M/S
- JSON/XML 模式隐藏这些参数
- 散图模式显示文件列表

### 4. 添加预览
- 选择图片后显示缩略图
- 切片模式下显示红色网格覆盖层
- 实时计算并显示预计帧数

### 5. 智能检测
- 选择 PNG 时检查同名 JSON/XML
- 如果有，自动推荐对应模式
- 提示用户可能的选择

## 实施建议

**Phase 1: 最小改进**
1. 改为居中模态弹窗
2. 流程改为：先选文件，再显示相关选项

**Phase 2: 增强体验**
1. 添加图片预览
2. 网格模式下显示切片预览
3. 智能检测文件类型

**Phase 3: 拖拽支持**
1. 支持拖拽文件导入
2. 批量导入优化
