# Blender 空状态行为研究 V2

## 用户需求

空状态（无精灵图集）时，左右两侧的边栏都应该隐藏，只保留中央画布和顶部 Header。

---

## Blender 的默认启动行为

### 启动时的界面布局

Blender 启动时的默认布局：
```
┌─────────────────────────────────────────────────────────────┐
│ 顶部菜单栏 (File, Edit, Render, Window, Help)                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                   Splash Screen / 3D Viewport               │
│                      "欢迎使用 Blender"                      │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 底部时间轴 (最小化)                                          │
└─────────────────────────────────────────────────────────────┘
```

**关键观察**：
1. **工具栏 (T-Panel)**: 默认 **隐藏** (按 T 键显示)
2. **属性面板 (N-Panel)**: 默认 **隐藏** (按 N 键显示)
3. **大纲视图**: 左侧可见，但可以通过 Ctrl+Space 最大化视图隐藏
4. **属性编辑器**: 右侧可见，显示场景级别的属性

### 3D 视口的空状态

当 3D 视口没有对象时：
```
┌────────────────────────────────────┐
│                                    │
│            3D Viewport             │
│                                    │
│     [网格地板] + [3D光标]          │
│                                    │
│   (没有工具栏，没有属性面板)        │
│                                    │
└────────────────────────────────────┘
```

- 工具栏: 隐藏
- 属性面板: 隐藏
- 只有按 T 或 N 才会显示

### 点击 "New" 后的变化

创建对象后：
```
┌─────────────────────────────────────────────────────────────┐
│ 顶部菜单栏                                                   │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│ 大纲视图  │          3D视图               │    属性编辑器      │
│ (可见)   │         [立方体]              │  (立方体属性)      │
│          │                              │                   │
├──────────┴──────────────────────────────┴───────────────────┤
│ 时间轴                                                       │
└─────────────────────────────────────────────────────────────┘
```

注意：即使创建对象后，**工具栏和属性面板仍然默认隐藏**，除非用户主动打开。

## 其他参考

### Unity

空项目启动时：
- Hierarchy: 显示 "None"
- Inspector: 显示 "Nothing selected"
- Project: 空
- Scene: 空的网格

所有面板都可见，但内容为空。

### Godot

空项目启动时：
- Scene dock: 显示 "No nodes in the scene"
- Inspector: 显示 "No object selected"
- FileSystem: 空

所有面板都可见，但显示空状态提示。

## 推荐的精灵编辑器空状态设计

### 方案 A: 纯 Blender 风格（推荐）

**空状态**:
```
┌──────────────────────────────────────────┐
│ 顶部菜单栏                                │
├──────────────────────────────────────────┤
│                                          │
│                                          │
│              画布区域                      │
│                                          │
│        "点击「导入」添加精灵图集"         │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│ 底部状态栏                                │
└──────────────────────────────────────────┘
```

- 左侧工具栏: 完全隐藏
- 右侧属性面板: 完全隐藏
- 中央画布: 显示引导信息
- 需要按快捷键 (如 T 或 N) 才会显示面板

**有图集后**:
```
┌──────────┬──────────────────────┬─────────┐
│          │                      │         │
│  工具栏   │       画布           │ 属性面板 │
│  (T)     │                      │  (N)    │
│          │                      │         │
└──────────┴──────────────────────┴─────────┘
```

- 默认显示工具栏和属性面板
- 或保持隐藏，让用户按快捷键打开

### 方案 B: 渐进式显示

**空状态**: 同方案 A

**有图集后**:
- 自动显示左侧工具栏
- 右侧属性面板保持隐藏，选择帧后才显示

---

## 实现建议

采用 **方案 A (纯 Blender 风格)**：

1. 空状态: 隐藏左右两侧边栏
2. 有图集后: 自动显示两侧边栏
3. 提供快捷键:
   - T: 切换工具栏
   - N: 切换属性面板

这样最符合 Blender 的设计哲学。
