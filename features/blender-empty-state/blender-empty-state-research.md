# Blender 空状态设计研究

## 当前问题

当编辑器刚开始没有精灵图集时，左侧工具栏显示"选择工具"和操作提示（左键选择、Ctrl多选等），但实际上没有内容可操作。这种提示显得多余，因为画布中央已经有"点击「导入」添加精灵图集"的提示。

---

## Blender 的空状态设计模式

### 1. 默认启动状态

Blender 启动时的布局：
```
┌─────────────────────────────────────────────────────────────┐
│ 顶部菜单栏                                                    │
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│ 大纲视图  │          3D视图               │    属性面板       │
│ (可见)   │      "点击打开文件"           │  (显示场景属性)   │
│          │                              │                   │
├──────────┴──────────────────────────────┴───────────────────┤
│ 时间轴 (可见但最小化)                                         │
└─────────────────────────────────────────────────────────────┘
```

**关键设计原则**：
- **工具栏（T-Panel）默认隐藏** - 按 T 键才会显示
- **属性面板（N-Panel）默认隐藏** - 按 N 键才会显示
- 视图区域显示启动画面或空状态提示
- 底部状态栏始终可见，显示基本提示

### 2. 没有选中对象时的工具栏行为

当场景中有对象但没有选中时：
- 工具栏显示当前模式的通用工具（如移动、旋转、缩放）
- 或者显示"无活动对象"（No active object）
- 许多工具处于禁用状态（灰色）

当场景中没有任何对象时：
- 工具栏要么隐藏，要么显示最小化的创建工具

### 3. 其他编辑器的空状态设计

#### Tiled (瓦片编辑器)
- 左侧工具栏始终显示，但在没有图块集时禁用
- 画布显示"拖入图片开始"
- 工具提示显示在鼠标悬停时

#### Unity Sprite Editor
- 没有资源时，编辑器无法打开（必须在 Project 窗口选择资源）
- 打开后，工具栏始终可见

#### Godot
- 没有选中节点时，Inspector 显示"选择一个节点以编辑属性"
- 工具栏（2D/3D视口顶部）始终可见但部分禁用

---

## 推荐解决方案

### 方案 1: 空状态隐藏工具栏（推荐）

当没有精灵图集时，左侧工具栏完全隐藏：
```
┌──────────────────────────────────────────┐
│ 顶部菜单栏                                │
├──────────────────────────────────────────┤
│                                          │
│              画布区域                      │
│        "点击「导入」添加精灵图集"         │
│                                          │
├──────────────────────────────────────────┤
│ 底部状态栏                                │
└──────────────────────────────────────────┘
```

当导入图集后，工具栏自动显示。

**优点**：
- 界面简洁，没有多余元素
- 用户专注于"导入"这个唯一操作
- 符合 Blender "默认隐藏工具栏"的原则

### 方案 2: 工具栏显示空状态提示

工具栏显示简洁的导入提示：
```
┌──────────┐
│ 选择模式  │
├──────────┤
│ 无图集   │
│          │
│ 点击顶部 │
│ 「导入」 │
│ 开始     │
│          │
├──────────┤
│ ☑ 显示   │
└──────────┘
```

**优点**：
- 布局稳定，不会因为导入而出现/消失
- 可以直接在工具栏提供"快速导入"按钮

### 方案 3: 操作提示改为 Tooltip

保留工具栏但移除固定的操作提示，改为：
- 鼠标悬停在"选择模式"按钮上时显示操作提示
- 或者按 F1 / ? 键显示帮助

```
┌──────────┐
│ 选择模式  │ ← 鼠标悬停显示工具提示
├──────────┤
│ ☑ 显示   │
└──────────┘
```

Tooltip 内容：
"选择模式: 左键选择, Ctrl多选, Shift范围选择"

---

## 最终推荐

**采用方案 1（空状态隐藏工具栏）+ 方案 3（Tooltip 提示）**：

1. 没有图集时，左侧工具栏隐藏
2. 有图集后，工具栏自动显示
3. 工具栏中的"选择工具"部分改为 Tooltip 提示
4. 在 Header 的模式按钮上添加 Tooltip 说明

这样既有简洁的空状态，又有良好的可发现性。
