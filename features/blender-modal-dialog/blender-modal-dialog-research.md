# Blender 的模态对话框设计研究

## 用户问题

居中模态弹窗是否符合 Blender 的设计原则？

## Blender 的设计哲学

Blender 的核心设计原则之一是：**最小化模态（Minimize Modality）**

但这并不意味着完全不用模态对话框，而是：
- 避免不必要的打断用户流程
- 重要的、需要用户确认的操作可以使用模态
- 优先使用非模态的面板和工具架

## Blender 使用模态对话框的场景

### 1. 文件操作（居中模态）

Blender 的文件浏览器是**居中模态对话框**：

```
┌─────────────────────────────────────────────────────────────────────┐
│  Blender File View                                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌────────────────────┐    ┌─────────────────────────────────────┐ │
│   │ 文件系统            │    │                                     │ │
│   │                    │    │         文件预览                     │ │
│   │ 📁 Documents       │    │                                     │ │
│   │ 📁 Desktop         │    │         [图片缩略图]                 │ │
│   │ 📁 Downloads       │    │                                     │ │
│   │   📄 model.fbx     │    │                                     │ │
│   │   📄 texture.png   │    │                                     │ │
│   └────────────────────┘    └─────────────────────────────────────┘ │
│                                                                     │
│   文件名: [model.fbx                    ]                           │
│                                                                     │
│                    [取消]  [打开/导入]                               │
└─────────────────────────────────────────────────────────────────────┘
```

**使用场景：**
- File > Open（打开文件）
- File > Import > FBX/OBJ/GLTF（导入）
- File > Export > FBX/OBJ/GLTF（导出）
- File > Save As（另存为）

**特点：**
- 居中显示
- 背景半透明遮罩
- 模态：必须完成或取消才能回到主界面
- 支持文件预览

### 2. 偏好设置（居中模态）

Edit > Preferences：
- 居中模态窗口
- 左侧分类导航，右侧详细设置
- 必须点击关闭或按 Esc 退出

### 3. 确认对话框

- 删除对象时的确认
- 退出未保存时的确认
- 覆盖文件的确认

### 4. 不使用模态的场景

Blender **不使用**模态对话框的场景（使用非模态面板替代）：

| 场景 | Blender 的做法 |
|------|----------------|
| 调整物体参数 | 右侧 Properties 面板 (N-Panel) |
| 选择工具 | 左侧 Toolbar (T-Panel) |
| 变换操作 | 底部状态栏 + 视口交互 |
| 渲染设置 | 右侧 Properties 的 Output 面板 |
| 材质编辑 | 独立的 Shader Editor 工作区 |

## 对比分析

### 精灵图导入 vs Blender 导入

| 对比项 | Blender 导入 | 当前精灵编辑器导入 | 优化建议 |
|--------|--------------|-------------------|----------|
| 触发方式 | File > Import | 顶部"导入"按钮 | ✅ 类似 |
| 对话框位置 | 居中模态 | 顶部展开 | ❌ 应改为居中模态 |
| 流程 | 选择文件 → 设置参数 → 导入 | 选择模式 → 选择文件 → 参数 → 导入 | ⚠️ 应改为先选文件 |
| 预览 | 右侧预览面板 | 无预览 | ⚠️ 应添加预览 |
| 参数位置 | 右侧参数面板 | 混合在弹窗中 | ⚠️ 应分离预览和参数 |

## 结论

### ✅ 居中模态弹窗符合 Blender 设计原则

**理由：**

1. **文件相关操作使用模态**是 Blender 的标准做法
2. **居中显示**符合 Blender 文件浏览器的设计
3. **半透明遮罩**可以提示用户这是模态操作
4. 导入是**重要的、需要确认的操作**，适合使用模态

### ⚠️ 需要注意的设计细节

要让模态弹窗**真正**符合 Blender 风格，还需要：

1. **左侧文件导航 + 右侧预览布局**
   ```
   ┌────────────────────────────────────┐
   │ 导入精灵图                          │
   ├────────────┬───────────────────────┤
   │ 文件列表    │  图片预览              │
   │            │  [显示切片网格]        │
   │            │                       │
   │            │  参数                 │
   │            │  W: 16 H: 16...       │
   └────────────┴───────────────────────┘
   ```

2. **非向导式的单面板设计**
   - Blender 倾向于单面板，而不是多步向导
   - 所有选项在一个面板内完成
   - 参数变化实时反映在预览上

3. **可停靠的导入设置**
   - 如果导入是常用操作，可以考虑像 Blender 的"导入设置"那样
   - 在左侧显示参数，右侧显示预览

## 推荐的 Blender 风格导入设计

### 方案 A: 纯 Blender 风格（推荐）

```
┌─────────────────────────────────────────────────────────────────────┐
│ 导入精灵图                                         [X]               │
├─────────────────────┬───────────────────────────────────────────────┤
│                     │                                               │
│  📁 assets/         │  ┌─────────────────────────────────────────┐  │
│  📄 player.png     │  │                                         │  │
│  📄 tiles.png  ◄───┼──││         预览区域                         │  │
│  📄 enemies.png    │  │                                         │  │
│                     │  │    [图片显示，红色网格覆盖层]            │  │
│  导入方式:          │  │                                         │  │
│  ● 网格            │  │    网格: 12×8 帧                         │  │
│  ○ JSON            │  │                                         │  │
│  ○ XML             │  └─────────────────────────────────────────┘  │
│  ○ 散图            │                                               │
│                     │  网格参数:                                     │
│  参数:              │  W: [16]  H: [16]  M: [0]  S: [0]            │
│                     │                                               │
│                     │              [取消]  [导入]                   │
│                     │                                               │
└─────────────────────┴───────────────────────────────────────────────┘
```

**特点：**
- 单面板，非向导式
- 左侧文件选择 + 模式 + 参数
- 右侧大预览区域
- 参数变化实时更新预览

## 最终建议

**居中模态弹窗是符合 Blender 设计原则的**，但应该：

1. ✅ 使用居中模态（符合 Blender 文件浏览器）
2. ⚠️ 避免向导式多步骤（Blender 偏好单面板）
3. ✅ 添加图片预览区域
4. ✅ 左侧/上方放参数，右侧/下方放大预览
5. ✅ 参数变化实时反映在预览上

**不推荐向导式的原因：**
- Blender 很少使用向导式流程
- Blender 用户偏好"所见即所得"的单面板
- 向导式在 Blender 中会显得不够直接
