# 精灵编辑器帧数据编辑方式调研

## 当前问题

当前使用右键菜单添加碰撞体存在以下问题：
1. **违反Blender原则**：Blender强调模式化编辑和属性面板，而非上下文菜单
2. **可发现性差**：用户必须右键才能发现功能
3. **批量编辑困难**：无法同时对多个帧设置碰撞体
4. **缺乏可视化反馈**：碰撞体形状在编辑前不可见

---

## 各编辑器设计原则分析

### 1. Blender 设计原则

**核心理念**：
- **模式化编辑 (Modes)**：Object Mode ↔ Edit Mode ↔ Sculpt Mode 等
- **属性面板 (Properties)**：右侧 N-Panel 显示选中对象的详细属性
- **工具栏 (Toolbar)**：左侧 T-Panel 提供当前模式的上下文工具
- **直接操作**：在3D视图中直接操作，而非通过菜单
- **非破坏性**：修改器栈、数据块引用

**相关模式**：
- 进入 Edit Mode 后，整个界面上下文改变
- 顶部菜单栏显示当前模式
- 工具栏根据模式动态变化

### 2. Tiled 图块编辑器

**碰撞体编辑方式**：
- **独立面板**：Tile Collision Editor（独立窗口/面板）
- **可视化绘制**：在图块上直接绘制碰撞形状
- **工具栏**：提供矩形、椭圆、多边形、擦除工具
- **对象类型**：支持设置对象类型属性

**特点**：
- 双击图块或点击"编辑碰撞"按钮进入编辑状态
- 左侧面板显示工具（矩形、圆形、多边形）
- 右侧或底部显示属性

### 3. Godot 引擎

**Sprite/TextureRegion 编辑**：
- **底部面板**：选中资源后，底部出现专用编辑器
- **Region 模式**：直接拖动选择区域
- **Collision 模式**：
  - 选中 Sprite 节点
  - 在 Inspector 中添加 CollisionShape2D 子节点
  - 在 2D 视图中直接调整形状

**特点**：
- Inspector 驱动（属性面板）
- 视口直接操作
- 节点化思维

### 4. Unity 2D (Sprite Editor)

**Sprite Editor 窗口**：
- **专用窗口**：双击 Sprite 资源打开 Sprite Editor
- **下拉模式切换**：
  - Single/Multiple Sprite 模式
  - **Custom Physics Shape** 模式专门编辑碰撞
- **可视化编辑**：在 Sprite 上绘制多边形点
- **属性面板**：右侧显示 Sprite 属性

**特点**：
- 资源级别的专用编辑器
- 模式切换（Slice → Edit → Physics Shape）
- 可视化拖动顶点

### 5. LDtk (现代图块编辑器)

**设计哲学**：
- **即时反馈**：所有操作立即可见
- **最小化模态**：减少弹窗和模式切换
- **智能默认值**：自动推断常见设置

**碰撞编辑**：
- 图块定义面板中直接显示碰撞预览
- 一键切换预设（空、满、斜坡）
- 支持自定义多边形

## 推荐方案：混合模式设计

基于以上分析，推荐采用 **Blender模式切换 + 可视化编辑 + 属性面板** 的混合方案：

### 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│  顶部工具栏                                                    │
│  [选择模式 ▼] [碰撞模式 ▼] [标签模式 ▼]  ← 模式切换 (Blender风格) │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    画布区域                                   │
│              ┌─────────────┐                                │
│              │  ┌───────┐  │  ← 选中帧显示碰撞体覆盖层          │
│              │  │ 帧图像 │  │                                │
│              │  │ ▓▓▓▓▓ │  │  ← 碰撞体可视化 (LDtk风格)        │
│              │  └───────┘  │                                │
│              └─────────────┘                                │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  底部状态栏：当前模式 | 选中: 3帧 | 提示: 点击添加矩形碰撞体       │
└─────────────────────────────────────────────────────────────┘
```

右侧/左侧 Inspector 面板：
```
┌────────────────┐
│ 帧属性          │
├────────────────┤
│ 名称: frame_12 │
│ 尺寸: 16×16    │
├────────────────┤
│ 碰撞体 ▼        │
│ ○ 无           │
│ ● 完整矩形      │ ← 单选按钮组
│ ○ 上半部分      │
│ ○ 斜坡 NE      │
│ ○ 自定义...    │
├────────────────┤
│ 标签            │
│ [wall][solid]  │
│ + 添加标签      │
└────────────────┘
```

### 模式设计（Blender风格）

#### 模式 1: 选择模式 (Select Mode)
- 默认模式
- 可以选择/多选帧
- 显示帧信息和缩略图
- 顶部工具栏切换其他模式

#### 模式 2: 碰撞编辑模式 (Collider Mode)
- **激活方式**：选中帧后点击工具栏按钮，或按 Tab 切换
- **画布行为**：
  - 显示碰撞体覆盖层（红色半透明）
  - 鼠标悬停显示预览
  - 单击应用当前工具的形状
- **工具栏**：
  - 矩形工具
  - 圆形工具
  - 多边形工具
  - 预设快速按钮（满、半、斜坡）
  - 擦除工具

#### 模式 3: 标签编辑模式 (Tag Mode)
- 批量为选中帧添加/移除标签
- 画布显示标签图标

### 交互流程对比

| 操作 | 当前方案 | 推荐方案 |
|------|----------|----------|
| 添加碰撞体 | 右键 → 选择预设 | 选中帧 → 点击"碰撞模式" → 点击帧/选择预设 |
| 批量添加 | 不可行 | 多选帧 → 点击预设 → 应用到所有 |
| 修改形状 | 不可行 | 碰撞模式 → 拖动顶点编辑 |
| 查看碰撞体 | 无 | 始终显示红色覆盖层（可开关） |

### 技术实现建议

1. **状态机设计**：
```typescript
type EditorMode = 'select' | 'collider' | 'tag';
type ColliderTool = 'rect' | 'circle' | 'polygon' | 'eraser';

interface EditorState {
  mode: EditorMode;
  colliderTool: ColliderTool;
  selectedFrames: string[];
}
```

2. **Inspector 面板**：
- 始终显示（类似 Blender N-Panel）
- 根据选中和模式动态更新内容
- 支持批量编辑（多选时显示"混合值"或"应用到所有"）

3. **可视化反馈**：
- 碰撞编辑模式下，画布显示半透明覆盖层
- 选中帧高亮边框
- 工具提示显示操作说明

4. **快捷键**：
- `Tab` - 切换模式
- `1/2/3` - 快速选择工具
- `G` - 抓取/移动（Blender风格）
- `X` - 删除碰撞体

## 参考截图/描述

## 实施优先级

1. **Phase 1**：Inspector 面板 + 简单属性编辑（替换右键菜单）
2. **Phase 2**：碰撞编辑模式 + 预设工具栏
3. **Phase 3**：可视化编辑（拖动顶点、直接绘制）
4. **Phase 4**：高级功能（批量编辑、自定义多边形）
